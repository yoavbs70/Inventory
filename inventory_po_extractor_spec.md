# Inventory PO Extractor — Spec

## Purpose
Filter inventory transactions by catalog and date range, extract PO numbers from the Notes column, and export results to a new Excel file.

## Project Files

| File | Description |
|------|-------------|
| `inventory_po_extractor.py` | Main Python script — interactive CLI for filtering and exporting |
| `inventory_po_extractor_spec.md` | This spec |
| `InventoryTransactionsF6.xlsx` | Source data file, synced daily from SharePoint |
| `InventoryTransactionsF6-1.xlsx` | Backup / alternate version of the source file |

## Source File Sync

The source file `InventoryTransactionsF6.xlsx` is downloaded from SharePoint using the Microsoft Graph API.

### Authentication
- Uses **OAuth 2.0 Device Code Flow** — on first run, a URL and code are printed; the user authenticates in a browser.
- The refresh token is saved encrypted to `%APPDATA%\sync-inventory-token.enc` using **Windows DPAPI** (`ConvertFrom-SecureString`).
- On subsequent runs the refresh token is used silently; re-authentication only happens if the refresh fails.

### Sync Script
- **`sync-inventory.ps1`** — downloads the file directly from a SharePoint drive item via Graph API.
  - Parameters: `-OutputFolder` (default: script directory)
  - Logs all activity to `sync-inventory.log`
  - Graph drive ID and item ID are hardcoded in the script
- **`sync-inventory.bat`** — double-click wrapper that runs the `.ps1` with `ExecutionPolicy Bypass`
- The sync runs **daily at ~09:00** (scheduled task on the local machine, based on log history)

## Extractor Workflow

### Step 1 — Ask for Catalog
Prompt the user to enter a Catalog value to filter on.

### Step 2 — Ask for Date Range
Prompt for:
- **From date** (required)
- **To date** (optional, defaults to today)

### Step 3 — Filter & Extract PO Numbers
- Filter rows where:
  - **Column C** (Catalog) matches user input (case-insensitive)
  - **Column I** (Transaction Date) falls within the specified date range
- Search **column G ("Notes")** for a PO number matching the pattern:
  - One or more uppercase letters, followed by `-` or `.`, then 5+ digits
  - Examples:
    - `P-5783539-E (ETX-2I-10G-DC-PS-C(1))` → PO = `5783539`
    - `PO.5778978(E)` → PO = `5778978`
    - `ND-5783728-E(ETX-2...)` → PO = `5783728`
    - `R-5783751-E(ETX2I-...)` → PO = `5783751`
  - Regex: `[PN][A-Z]*[-.](\d{5,})` → must start with P or N; capture group 1 is the numeric PO
- Rows where column G does not match this pattern are excluded.

### Step 4 — Sort Results
Order the filtered rows by **Transaction Date (column I)** ascending, then by **extracted PO number** (numeric) ascending.

### Step 5 — Export
Create a new Excel file in the same directory as the source file:
- Filename: `PO_Results_<Catalog>_<FromDate>_<ToDate>.xlsx`
- Include all original columns plus an additional column **"PO Number"** containing the extracted numeric PO.
- Column mapping reference:
  - **C** — Catalog
  - **G** — Notes (PO source)
  - **I** — Transaction Date (filter + sort key)

## Notes
- Console/terminal UI only — no GUI.
- Use `openpyxl` for reading/writing Excel files.
- Date comparison should be robust to both date-only and datetime values in the source file.
- The sync script requires no additional PowerShell modules — it uses only built-in `Invoke-RestMethod` and `Invoke-WebRequest`.
- Before running the extractor, ensure the source file is up to date by running `sync-inventory.bat` (or the `.ps1` directly).
